# 全局阶段计划

## Gate 0：Spec 审阅

目标：人工审阅并确认全局内置指标扩展协议。

验收条件：

1. 确认项目正式回测链路只允许使用 Rust 内置指标。
2. 确认不引入运行时外部自定义指标。
3. 确认所有内置指标默认对齐原版 `pandas-ta`，除非原版缺失。
4. 确认 task 子目录结构可用于后续长期扩展。

未通过本 Gate 前不得进入代码实现。

## Gate 1：AI pre-review

目标：执行前检查任务文档与当前源码范式是否一致。

验收条件：

1. 审阅报告按 `文档清单 -> 总体判断 -> Findings` 输出。
2. 明确审阅 `00_meta / 01_context / 02_spec / 03_execution`。
3. 明确核对核心指标目录、registry、指标测试模板和 warmup contract 测试范式。
4. 若文档与源码不一致，先修文档，不直接改代码。

## 阶段 1：落地单指标

目标：按某个 `02_spec/indicators/<base_name>/` 的规格实现该指标。

验收条件：

1. 指标进入 Rust 内置指标系统。
2. 指标通过 registry 注册。
3. 输出列与单指标 Spec 一致。
4. 不存在外部运行时指标实现。

## 阶段 2：补测试

目标：新增或更新该指标的准确性测试和 warmup contract 测试。

验收条件：

1. 对齐正式真值源。
2. 覆盖参数和输出列。
3. 覆盖 warmup 和 mode。

## 阶段 3：验证

目标：按仓库统一入口执行检查。

验收条件：

1. `just check` 通过。
2. 单指标测试通过。
3. warmup contract 测试通过。

## 阶段 4：post-review

目标：确认实现、测试和文档一致。

验收条件：

1. post-review 无阻断问题。
2. 旧痕迹扫描完成。
3. 单指标 `04_review/indicators/<base_name>/` 记录执行结果。
