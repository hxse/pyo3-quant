# 任务元信息

## 1. 任务描述

`2026-04-16 extensible builtin indicators` 是内置指标体系的长期扩展任务。项目默认且唯一正式路径是：策略与回测配置只声明指标参数和信号模板，指标计算必须由 Rust 内置指标系统完成，信号模板只能消费内置指标输出列。

本任务以 `kc` 肯特纳通道作为首个新增指标实例，同时冻结后续新增类似指标时必须复用的长期扩展协议。后续新增指标应在本任务的子目录中补充单指标规格和执行记录，而不是重新讨论项目级口径。

## 2. 任务级别

本任务定为 `A 类任务`。

原因：

1. 内置指标属于 Rust 回测引擎的核心计算真值，新增指标会进入公共 `indicators_params` 契约、输出列契约、warmup 契约和信号模板消费链路。
2. 本任务冻结的是长期扩展口径，不只是单个 `kc` 指标；若协议不清，后续指标容易出现命名漂移、公式漂移、测试基准漂移和 warmup 漂移。
3. 指标 quietly wrong 风险高。错误公式、错误预热、错误输出列或错误真值源都会让回测表面正常但信号语义错误。
4. 任务涉及 Rust 指标模块、注册表、Python 指标测试、warmup contract 测试和类型存根同步，因此需要完整 Spec Gate、Execution Gate、Stage Gate 与 Close Gate。

## 3. 任务范围

### 范围内

1. 冻结项目级内置指标扩展协议：只能使用 Rust 内置指标，不允许运行时外部自定义指标。
2. 冻结新增指标的文件结构、注册方式、参数命名、输出列命名、warmup 契约、测试基准和验证入口。
3. 设计可长期追加的 task 子目录结构，使后续每个指标都有独立规格、执行计划和验收记录。
4. 以 `kc` 作为首个指标实例，定义其公式、参数、输出列、失败语义、测试口径与执行计划。
5. 明确所有内置指标默认以原版 `pandas-ta` 对齐；仅当原版 `pandas-ta` 缺失该指标时，才允许按项目指标优先级后退。
6. 后续实现阶段必须使用仓库正式验证入口，包括 `just check` 和指标测试入口。

### 范围外

1. 本轮只更新 task 文档，不实现代码。
2. 本任务不新增运行时外部指标系统、动态插件系统、Python 自定义指标接口或临时外部计算通道。
3. 本任务不修改已有指标公式、输出列或 warmup 契约；如发现已有指标存在偏差，应拆成单独指标修复记录或后续子任务。
4. 本任务不为新增指标提供旧名、别名、兼容参数或双轨输出列。
5. 本任务不改变现有 `indicator_key.split('_').next()` 的注册分发模型。
