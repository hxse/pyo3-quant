# KC Breaking Changes

## 1. 正式判断

KC 是新增内置指标，不删除已有公开指标，不修改已有指标输出列，不改变现有信号模板语法。

因此对已存在用户代码不构成破坏性变更。

## 2. 有意收紧

KC 从第一版开始禁止以下双轨口径：

1. 不提供 `period` 参数别名。
2. 不同时输出 `basis` 和 `middle`。
3. 不开放 `tr` 或 `mamode` 参数。
4. 不允许用当前独立 `atr` 指标近似替代 `EMA(True Range)`。
5. 不允许运行时外部计算 KC 后注入回测链路。

## 3. 后续扩展

若未来确实需要支持 `tr=false` 或其他 `mamode`，必须作为新指标规格变更处理，并补充对齐测试和影响范围。
