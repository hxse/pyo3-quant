# 方案取舍

本任务的核心取舍可以压成四条：

1. 选择 breaking update，而不是继续保留旧桥接与旧兼容。
2. 选择单一真值链，而不是在多个容器、多个包装层重复维护平行语义。
3. 选择 fail-fast 与显式 contract，而不是依赖静默回退或弱约定。
4. 选择先冻结结构真值与运行边界，再进入代码落地，而不是边写边猜。

这四条取舍决定了 `03-10` 文档整体的写法：

1. `02_spec` 负责把正式设计状态讲清楚。
2. `03_execution` 负责把落地顺序、测试与 cleanup 收口写清楚。
3. 旧逻辑清理不被当成“后续优化”，而被视为正式落地的一部分。

本页不重复展开对象 contract，具体真值以 `02_spec` 正文为准。
