# 项目内置指标口径

## 1. 正式路径

项目默认走模板化回测引擎。策略与回测配置只声明：

1. 数据源。
2. 内置指标参数。
3. 信号模板。
4. 回测参数。

指标计算必须由 Rust 内置指标系统完成。信号模板只能引用数据列和 Rust 内置指标输出列。

## 2. 禁止外部自定义指标

运行时不允许外部自定义指标参与正式回测链路。

禁止形式包括：

1. 在 Python 策略层临时计算指标列。
2. 在 Python 工具层计算指标后注入信号模板。
3. 在回测链路外部用 pandas、pandas-ta、numpy 或其他库生成运行时指标列。
4. 为某个策略单独维护一套与 Rust 内置指标平行的指标计算逻辑。

Python 可以在测试中调用基准库，用于验证 Rust 内置指标是否对齐正式真值源。

## 3. 原因

该口径已经是项目基础取舍，不在本任务重新辩论。

原因：

1. Rust 是核心计算真值来源。
2. 内置指标可以统一进入 `indicators_params`、warmup contract、信号模板解析和结果导出链路。
3. 外部运行时指标会绕过契约聚合和统一验证，容易 quietly wrong。
4. 单一指标真值能避免同名指标在不同策略中出现不同公式、不同预热和不同空值语义。

## 4. 新增指标路径

当策略需要新指标时，必须先把该指标加入 Rust 内置指标系统，再通过模板化链路引用其输出列。

新增指标不应作为策略局部实现出现。
