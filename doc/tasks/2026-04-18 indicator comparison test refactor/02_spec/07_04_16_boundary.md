# 与 2026-04-16 的边界

## 1. 顺序关系

本任务应先于 `2026-04-16 extensible builtin indicators` 的代码实现执行。

原因：

1. 04-16 会新增 KC 指标。
2. KC 测试应直接落在新指标比较系统上。
3. 避免先写旧模板测试，再迁移到新模板时混淆指标实现偏差与测试系统偏差。

## 2. 本任务不修改 04-16 指标规格

本任务不改变 04-16 中对内置指标扩展、KC 参数、KC 输出列、KC warmup 的定义。

本任务只定义测试系统口径。

## 3. 04-16 后续需要调整的内容

本任务完成后，04-16 至少需要更新：

1. 全局验证策略。
2. KC 测试契约。
3. KC 测试目录结构，使用 `py_entry/Test/indicators/kc/` 高内聚目录。
4. 后续新增指标的测试模板引用方式。

04-16 不应再引用旧的 `IndicatorTestConfig + assert_mode_*` 作为长期测试口径。

04-16 不应新增平铺 `py_entry/Test/indicators/test_kc.py` 作为长期结构。
